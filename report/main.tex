\section{Part I: Inverse Problems in Imaging}

\subsection{A.1 Forward Problem: Simulating Image Acquisition}

The first stage models the acquisition process for a diffraction-limited optical
instrument. Starting from a clean microscopy image \(g \in \mathbb{R}^{m \times n}\),
the observed image is generated as
\begin{equation}
    f = H g + w,
\end{equation}
where \(H\) is a point-spread-function blur operator and \(w\) is additive
Gaussian noise. The implementation normalises the selected TIFF frame to
floating point intensities in \([0,1]\), applies a Gaussian PSF with standard
deviation \(\sigma = 2.0\), and then adds zero-mean Gaussian noise with
\(\sigma_n = 0.001\). The random number generator is seeded with 42 by default,
so the simulated acquisition is reproducible.

The default command downloads the PhC-C2DH-U373 microscopy dataset, selects the
first sorted frame from sequence 01 as the ground truth image, and writes the
ground truth, blurred image, blurred noisy image, comparison figure, and metadata
to \texttt{outputs/part\_i/a1/}.

\subsection{A.2 Classical Image Restoration}

The second stage treats the simulated observation from Part I.A.1 as the input
for classical restoration. I implemented isotropic total variation denoising by
approximately solving
\begin{equation}
    \min_g \frac{1}{2}\lVert g - f \rVert_2^2 + \lambda\,\mathrm{TV}(g),
\end{equation}
where the discrete TV term is computed from forward finite differences in the
horizontal and vertical directions. The optimisation is solved with a
Chambolle--Pock primal-dual method. The dual variables are projected onto the
pointwise \(\ell_2\) ball of radius \(\lambda\), and the objective value is
recorded after each iteration.

I evaluate three regularisation strengths by default:
\(\lambda=0.001\), \(\lambda=0.01\), and \(\lambda=0.08\). These are intended to
represent under-regularisation, a moderate compromise, and over-regularisation.
Small \(\lambda\) values keep the solution close to the measured image, so fine
texture and noise are both retained. Larger \(\lambda\) values suppress noise
more aggressively but can flatten subtle structures. On the default frame, the
noisy blurred observation gives PSNR 33.91 dB and SSIM 0.9219. A Gaussian filter
with \(\sigma=1\) gives PSNR 33.35 dB and SSIM 0.9154, while TV denoising gives
PSNR/SSIM values of 33.86/0.9201, 33.51/0.9097, and 31.85/0.8842 for
\(\lambda=0.001\), \(0.01\), and \(0.08\), respectively. These values show that
the strongest regularisation is over-smoothing this example. TV regularisation
tends to preserve edges better than a standard Gaussian filter because the
penalty is on the magnitude of the image gradient rather than on local averaging
itself; sharp transitions can remain when they are supported by the data
fidelity term.

\subsection{B(i) Algorithm Unrolling}

For the data-driven reconstruction experiment, I implemented a learned unrolled
proximal-gradient network. The forward operator \(H\) is a Gaussian blur with an
\(11\times 11\) kernel and standard deviation \(2.0\). Training observations are
simulated as \(f = Hg + w\), with white Gaussian noise of variance \(0.001\). The
network uses \(K=6\) unrolled iterations by default:
\begin{equation}
    g^{(k+1)} = \phi_{\theta^{(k)}}\left(g^{(k)} - \eta^{(k)}H^\top(Hg^{(k)} - f)\right),
\end{equation}
where each \(\phi_{\theta^{(k)}}\) is a small CNN proximal map with two hidden
convolutional layers and PReLU activations. The step sizes are learned as
unconstrained parameters passed through a softplus transform, which keeps
\(\eta^{(k)}\geq 0\) during training. The training loss is the \(\ell_1\) error
between the final reconstruction \(g^{(K)}\) and the clean reference image.
Training uses random crops and random horizontal/vertical flips from sequence 01;
sequence 02 is used for test metrics.

The full GPU run used 50 epochs. On the test sequence, the degraded input scored PSNR 28.18 dB and SSIM 0.4897, while the unrolled reconstruction improved this to PSNR 34.10 dB and SSIM 0.9113.

\subsection{B(ii) Out-of-Distribution Generalisation}

For the OOD experiment the same independent-layer unrolled PGD architecture is
trained with a more severe degradation, using a Gaussian blur kernel of size
\(21\times21\), standard deviation \(4.0\), and white Gaussian noise of variance
\(0.001\). At test time the model is evaluated on the original Part I.B(i)
setting, namely an \(11\times11\) Gaussian kernel with standard deviation \(2.0\)
and the same noise variance. This isolates whether the learned proximal maps are
robust to a forward model mismatch. The script records average PSNR and SSIM for
both the degraded input and the reconstruction, plus a visual comparison. The
expected conclusion is that training on a stronger blur can make the learned
prior more aggressive, but a mismatch in \(H\) can reduce reconstruction quality
relative to the matched Part I.B(i) model if the learned updates overcompensate
for blur not present at test time.

The full OOD run achieved PSNR 33.11 dB and SSIM 0.9030 on the matched test degradation. This is lower than the matched B(i) model (34.10 dB and 0.9113), which supports the expected conclusion that a learned unrolled solver is sensitive to forward-model mismatch, even when the training degradation is more severe.

\subsection{B(iii) Shared-Parameter Convergence}

For the convergence experiment I train a shared-parameter unrolled PGD model:
\begin{equation}
    g^{(k+1)} = \psi_{\theta}\left(g^{(k)} - \eta H^\top(Hg^{(k)} - f)\right),
\end{equation}
with a single CNN proximal map and a single positive softplus-constrained step
size reused at every layer. Training uses \(K=6\) iterations under the matched
\(11\times11\), \(\sigma=2.0\) blur and noise variance \(0.001\). At evaluation,
the same learned update is applied to one test image for \(T=K,4K,8K,16K\). The
script writes a PSNR/SSIM table and a PSNR curve. If PSNR improves then plateaus,
the learned update behaves like a stable iterative solver; if it degrades for
large \(T\), the finite-depth training has not learned a globally stable fixed
point. The full run achieved PSNR/SSIM values of 33.61/0.9049 at \(T=K\), 23.88/0.8278 at \(4K\), 18.05/0.5773 at \(8K\), and 6.56/0.0068 at \(16K\). The shared update is useful at the trained depth but becomes unstable when repeatedly applied far beyond the training horizon.

\section{Part II: Motion Estimation and Cell Tracking}

\subsection{A. Motion Estimation}

I implemented Horn--Schunck optical flow for a consecutive pair from sequence
01. The pair is normalised to \([0,1]\), then spatial derivatives are estimated
from the mean of the two frames and the temporal derivative is \(D_t f=f_{t+1}-f_t\).
The implementation minimises the optical-flow constraint residual with a
Tikhonov smoothness term,
\begin{equation}
    E(u,v)=\langle (D_xf u+D_yf v+D_tf)^2\rangle + \alpha\langle
    |\nabla u|^2+|\nabla v|^2\rangle .
\end{equation}
Each Jacobi step uses the four-neighbour average for \(\bar u\) and \(\bar v\)
and the standard residual-correcting update. Accuracy is tested on synthetic
translated images by checking that the energy decreases and that the output
shape and direction are sensible. The absolute frame difference is used as a
simple change visualisation; it highlights locations where intracellular
contrast changes between frames. Larger \(\alpha\) gives smoother, lower-magnitude
flows, while more iterations reduce the energy until convergence or numerical
plateau. For the generated run, the energy decreased from \(1.859\times10^{-3}\) to \(1.842\times10^{-3}\).

\subsection{B. YOLO Cell Segmentation}

The segmentation labels in \texttt{01\_GT/SEG} and \texttt{02\_GT/SEG} are
converted to YOLO segmentation format by extracting each connected component,
finding its longest contour, simplifying the polygon, and normalising polygon
coordinates by image width and height. Sequence 01 is exported as the training
split and sequence 02 as validation, with a \texttt{data.yaml} file containing a
single class, \texttt{cell}. Export verification rasterises the YOLO polygons and
overlays them on the corresponding microscopy frame. The model training script
uses \texttt{ultralytics.YOLO}; after training, \texttt{model.val} reports the
segmentation mAP50--95(M). The model must learn whole-cell masks while ignoring
phase-contrast texture, background substrate structure, and partial/ambiguous
objects near frame boundaries. Predictions are converted back to label images by
filling each predicted polygon with \texttt{skimage.draw.polygon}. The trained YOLO model reached validation mAP50--95(M)=0.694.

\subsection{C. Cell Tracking}

For tracking, the trained segmentation model is applied to every frame in
sequence 01 to produce instance label images. Centroids, areas, frame indices,
and instance labels are extracted with \texttt{skimage.measure.regionprops}. A
tracking algorithm also needs temporal association information: the detections
alone do not identify which cell in frame \(t+1\) corresponds to a cell in frame
\(t\), and masks can be missing or merged. The tracking script uses
\texttt{laptrack.LapTrack} with squared Euclidean distance, a configurable
maximum displacement, and a one-frame gap-closing allowance. In environments
without \texttt{laptrack}, the tested nearest-neighbour fallback links detections
within the same distance threshold. Missing masks can be compensated by gap
closing, motion priors, and by allowing tracks to persist briefly without a
measurement. A direct way to relate tracks and optical flow is to average the
dense flow inside each cell mask and compare this local internal-motion vector
with the displacement vector of the cell centroid over the same interval. The final YOLO-derived tracking output contains 287 detections assigned to 28 tracks. The tracker can bridge short missing-mask gaps with its gap-closing allowance; longer gaps or merged masks remain the main source of track fragmentation.
